## 1. 文档说明

本文档汇总「个人品牌网站」的产品定位、目标用户、信息架构、功能范围、内容策略、合规边界、设计风格与技术约束，供设计与开发对齐使用。

---

## 2. 产品概述

### 2.1 产品名称与愿景（来自现有 README）

- **名称**：秘术云阁  
- **愿景**：打造全球华人精神网络家园（长期愿景；当前站点落地以「一人公司与创业陪跑」为主叙事）  
- **Slogan（可选用）**：认知破局 | 底层翻身 | 经验传承（与站内主叙事需统一，避免口号与页面正文冲突）

### 2.2 核心定位（已确认）

| 维度 | 结论 |
|------|------|
| 商业定位 | **一人公司产品化** + **创业陪跑**（双主线，产品化偏交付与工具，陪跑偏咨询与方法论） |
| 目标用户 | **偏向个人**（独立开发者、想启动一人公司的个体、需要轻量咨询与陪跑的用户） |
| 地域与语言 | **中文站点**，面向**中国大陆用户** |
| 主站与私域 | 主站需**兼顾**：① 导流到微信；② **公众号二维码**「扫一扫关注」 |

### 2.3 个人叙事主线（已确认）

从对大环境趋势的判断出发，认为未来**个人公司 / 一人主体协作**会成为重要生存与发展模式；本人**率先站出来**，带领大家一起探索**一人公司如何起步与发展**。  

**表述原则**：突出「探索、方法论、同行者」，避免与违规服务产生联想；不强调「革命」等易引发监管或平台敏感联想的措辞，可在文案中弱化为「转型」「实践」「同行探索」。

## 3. 目标与成功指标（建议）

### 3.1 业务目标（定性）

- 建立可信赖的个人品牌触点，使访客理解服务范围与合作方式。  
- 完成从「浏览 → 了解服务 → 联系 / 加微信 / 关注公众号」的转化路径。  
- 通过博客持续输出，支撑 SEO 与私域运营。

### 3.2 MVP 成功标准（可量化建议）

- MVP 上线后具备完整五类页面（见 §5），PC 与移动端可用，支持亮/暗色主题。  
- 联系与导流路径清晰可测（表单提交、微信引导、公众号展示位）。  
- 公开价目表与咨询档位一致（见 §4）。

---

## 4. 服务与定价（已确认）

### 4.1 公开价目

| 类型 | 价格 | 说明 |
|------|------|------|
| 按次咨询 | **199 元/次** | 公开标价 |
| 按月陪跑 | **599 元/月** | 公开标价 |
| 其他 | **询价** | 作为备选入口（如定制开发、深度项目等） |

### 4.2 服务呈现建议

- **服务页**需分块说明：创业陪跑（199/599 对应关系）、一人公司产品化（全栈交付范围简述）、询价项（范围与联系方式）。  
- 暂无对外案例时，用「服务说明、流程、交付物类型、响应方式」建立信任，并预留「案例」模块占位，有素材后再补。

### 4.3 合规与灰色业务（重要）

用户说明：历史上曾考虑的「绝版资源」等涉及**非 100% 合规**、**不宜主站展示**。  

**产品结论（必须遵守）**

- **主站、博客正文、SEO 可见文案**：**不**描述、不暗示违规资源交易；**不**作为卖点关键词。  
- **私域**：仅可表述为「进一步咨询可能涉及的方向由私下沟通」，**不在 PR/页面中展开操作细节**。  
- 开发与运营人员仅按「合规展示」实现站点；私域话术由运营自行负责且须自负合规风险。

## 5. 信息架构与 MVP 范围

### 5.1 MVP 必含页面（已确认）

1. **首页** — 价值主张、服务入口、信任要素（无案例时用方法论与承诺替代）、导流区（微信 + 公众号）。  
2. **服务** — 定价表、服务清单、询价说明。  
3. **联系** — 表单、微信引导、公众号二维码、可选补充联系方式（由你后续提供具体账号信息）。  
4. **博客** — 列表 + 文章详情（架构可先做占位与一篇示例）。  
5. **关于** — 个人叙事主线、合作原则、与「一人公司探索」相关的立场（合规表述）。

### 5.2 建议后续迭代（非 MVP 必选）

- 案例/作品墙（有项目后）。  
- 站内搜索、标签体系、RSS。  
- 预约日历（如后续引入）。

## 6. 功能需求摘要

| 模块 | 需求 |
|------|------|
| 响应式布局 | PC / 移动一致体验；**版心两侧留白**，阅读舒适。 |
| 主题 | **亮色 / 暗色**切换，全站一致。 |
| 导流 | 微信加好友路径 + 公众号二维码展示（位置：页脚、联系页、首页关键区块可重复）。 |
| 表单 | 联系页收集需求类型、预算意向、联系方式等（字段可在实现阶段细化）。 |
| 博客 | 支持文章列表与详情；便于后续接 CMS 或 Markdown。 |

### 6.1 不需要在站内展示的内容（已确认）

- **公司主体**、发票与合同资质等：**当前无**，**不需要在网站展示**。

## 7. 内容规划

### 7.1 首页

- 一句话价值：一人公司产品化 + 创业陪跑。  
- 三条以内核心能力（如：全栈交付、陪跑节奏、轻量可起步）。  
- 定价入口（跳转服务页或锚点）。  
- 微信 + 公众号区块。

### 7.2 服务页

- 199 / 599 与「包含什么、不适合谁」写清楚，降低纠纷预期。  
- 询价项：定制开发、小程序、自动化等（与 README 能力一致，表述合法合规）。

### 7.3 博客

- 方向：创业认知、一人公司方法、技术随笔与提效（与定位一致）。  
- 为公众号导流：文末或侧栏固定「更多内容见公众号」+ 二维码。

### 7.4 关于页

- 讲述「为何现在做一人公司」「如何陪跑与交付」的个人故事线。  
- 避免具体违规业务；强调边界与正规合作方式。

## 8. 设计风格（来自 README + 探讨结论）

- **整体**：高端、大气；**黑金**基调（深色底 + 金色/琥珀点缀），专业、可信赖。  
- **个性化**：通过字体层级、少量几何/装饰元素、统一组件风格体现品牌，避免廉价渐变堆砌。  
- **可读性**：暗色模式注意对比度与正文行宽，避免纯黑底 + 刺眼金字。

---

## 9. 技术约束（来自 README）

- **框架**：Nuxt 4  
- **样式**：Tailwind CSS 4、SCSS  
- **语言**：TypeScript  
- **组件**：Nuxt UI 4、Element Plus（按需使用，避免两套组件视觉打架，建议以 Tailwind token 统一皮肤）  
- **实现要求**：以 Nuxt `pages` 路由组织页面；响应式与主题切换为硬性要求。

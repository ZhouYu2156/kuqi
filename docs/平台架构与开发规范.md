# 平台架构与开发规范（音乐 + 视频教程 + 知识付费）

## 1. 产品方向

- **内容**：音乐模块（酷狗搜索 + 播放）、个人视频教程（`/learn` 占位）、博客与品牌页。
- **用户身份**（`shared/types/auth.ts` · `UserRole`）：
  - **guest**：随便注册看看，低黏性。
  - **user**：不开会员但常访问（白嫖黏性较好）。
  - **vip**：铁杆粉丝，黏性最高。
  - **admin**：管理员；使用独立后台入口（`/admin`，占位扩展用户管理等），与普通用户 UI 区分。
- **鉴权范围（当前实现）**：**仅在「获取音乐播放地址」时做强校验**（`POST /api/music?audioId=`）。搜索列表 `GET /api/music` 可不登录。其他页面路由不强制登录。
- **音乐额度**：非无限用户 **每日免费试听 10 首**（`server/lib/musicQuota.ts` · `DAILY_FREE_LIMIT`）。超出返回 `403`，业务码见 `ResponseCode.Forbidden`，`data` 可含 `reason: 'MUSIC_QUOTA'`，前端引导 `/membership`。
- **无限试听**：`admin`、`vip`，或 `membership_until` 仍有效（视为付费在期）时 **不计次**。后续若需与「仅 vip + 指定 plan」严格对齐，可在 `hasUnlimitedMusic` 中收紧。
- **会员产品（文案）**：
  - **月度 ¥12**：听音乐、下载、看视频教程。
  - **年度**：包含月度全部权益，另含思维成长、认知提升等扩展资料（定价与交付以实际上线为准）。
- **UI**：全站统一 **`JkShell`**（`jk` / `default` 布局均挂载同一壳层）。
  - **主色**：`useJkTheme` / `app/composables/useJkTheme.ts` 中 **`JK_ACCENT_PRESETS`**（含 Nuxt 翠、祖母绿、天蓝、紫罗兰，以及 **品红、深粉、金黄、琥珀、红紫、洋红、玫红、珊瑚** 等），写入 CSS 变量 `--jk-primary`、`--jk-primary-soft`、`--jk-ring`。右上角 **`JkThemeControls`** 弹出面板可滚动，选项较多时纵向滚动浏览。
  - **暗色背景**：`assets/css/main.css` 的 `.jk-theme` 定义 **`--jk-bg-dark`（`#16101f`，暗系紫底）**、**`--jk-bg-dark-elevated`（`#1e1628`，略抬高表面）**。浅色页为 `bg-zinc-50`，暗色为 **整页暗紫底**（非纯灰黑）；顶栏、卡片、登录卡片等使用 `color-mix` 或变量与主色协调。
  - **明暗切换**：`UColorModeButton` + `nuxt.config` 的 `colorMode`（`preference: 'system'` 等）。
  - **注册验证码**：`AuthOtpDigits` 为 **左对齐**（`justify-start`），与「邮箱验证码」标签、上方邮箱输入左缘对齐；「发送验证码」为原生 **`<button>`**，背景 **`var(--jk-primary)`** 与悬停 `brightness`。**登录 / 注册页** 容器类名 **`auth-page`**：为 Element 设置 `--el-input-focus-border-color` 等以跟随主题；暗色下 **`--el-input-bg-color`** 使用 **`--jk-input-surface-dark`**（见 `main.css`），避免输入区死黑。顶栏「登录」链接同主色并带 **`hover:brightness-110`**。

## 2. 鉴权与数据

| 项目 | 说明 |
|------|------|
| 凭证 | **JWT（HS256）**，**HttpOnly Cookie `jk_auth`**；亦支持 `Authorization: Bearer`。 |
| 载荷 | `sub`、`email`、`role`、`membershipUntil`、`membershipPlan`（`monthly` \| `annual` \| `null`） |
| 签发 | `server/utils/jwt.ts`（`jose`） |
| 解析 | `server/utils/authContext.ts` → `getAuthUser` |
| 登录 | `POST /api/auth/login` |
| 注册 | `POST /api/auth/register`（邮箱 + 密码 + 4 位邮箱验证码；验证码由 `POST /api/auth/send-register-code` 发送，生产需接邮件服务） |
| 当前用户 | `GET /api/auth/me` |
| 退出 | `POST /api/auth/logout` |
| 音乐额度 | `GET /api/user/music-quota`（需登录，供音乐页展示剩余次数） |

**数据库**：

- 新环境：执行 `server/db/migrations/001_users.sql`（含 `membership_plan`）+ `002_music_quota_and_membership.sql` 中的 **`user_music_daily` 建表**（若 `001` 已含 `membership_plan`，可按注释跳过 `002` 里重复的 `ALTER`）。
- 自旧库升级：按 `002` 中顺序执行（含 `creator` → `user` 迁移）。

**开发免库登录**：`.env` 中 `AUTH_DEV_MODE=true` 与 `DEV_AUTH_EMAIL` / `DEV_AUTH_PASSWORD`（见 `login.post.ts`）。**生产**务必设置 `JWT_SECRET`（≥16 字符）。

## 3. 统一请求 / 响应

结构见 `shared/types/common.ts`：`{ code: ResponseCode, message, data, requestId? }`。

- 服务端：`server/utils/apiResponse.ts` → `ok` / `fail`。
- 音乐播放：`server/api/music/index.post.ts`（额度 → `getSingleMusic` → 非无限则 `incrementTodayPlay`）。
- 前端：`useAuth()` + `plugins/auth.client.ts` 首屏拉取；音乐播放与额度接口需 **`credentials: 'include'`**（见 `JKKugouSearch.vue`）。

## 4. 布局与路由

| 布局 | 用途 |
|------|------|
| `jk` | 首页、音乐、视频课、登录、会员说明、后台等：`JKNav` + `.jk-theme` |
| `default` | 与 `jk` 相同（均为 `JkShell`），未指定布局的页面默认走 `default` |

- `/admin`：`middleware: admin`，`nuxt.config` 中 **`/admin/**` 关闭 SSR**，避免首屏无 Cookie 态误判；管理页为占位，后续接用户列表与订单。
- `/membership`：会员权益与定价说明（支付对接前占位）。

## 5. 后续扩展

- 支付回调：更新 `users.membership_until`、`membership_plan`，并考虑短 TTL JWT 或刷新会话。
- 视频课：`requireRole` 保护 `server/api/course/*`。
- 管理后台：用户列表、改角色、改会员到期、审计日志。
